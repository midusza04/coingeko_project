app.py — main functions
app.py — database + ETL + UI main module
create_database() Creates 3 tables, indexes, and seeds 5 coins. Called at startup. database
fetch_market_chart() GET /coins/{id}/market_chart — fetches 365-day history. api
store_market_chart() INSERT OR REPLACE into market_snapshots — idempotent historical write. etl
fetch_markets_current() GET /coins/markets — current snapshot for 5 coins. api
store_current() INSERT into market_current — append-only, with collected_at UTC. etl
load_snapshots() SELECT + JOIN → pandas DataFrame. Cache 60s (@st.cache_data). read
page_*() × 6 Overview · Data Collection · Time Series · Quantitative · Dashboard · Correlation ui
Database Schema (crypto_market.db)
cryptocurrencies id TEXT PK · symbol · name (5 rows)
market_snapshots record_id PK · crypto_id FK · snapshot_date · price_usd · market_cap · total_volume
  UNIQUE(crypto_id, snapshot_date) · INDEX idx_snapshots_date (~1,826 rows)
market_current record_id PK · crypto_id FK · collected_at · price · mcap · high_24h · low_24h · changes · rank · supply · ath
  INDEX idx_current_collected_at (append-only)
