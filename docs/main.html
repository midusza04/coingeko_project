Moduł main

Warstwa ingestion danych rynkowych kryptowalut z CoinGecko API. Pobiera notowania Bitcoina, Ethereum i Solany i zapisuje odpowiedź jako sformatowany plik JSON.

📄 main.py 🐍 Python ≥ 3.14 📦 requests ≥ 2.33.1 🌐 CoinGecko API v3
📝

Opis modułu

Moduł pobiera aktualne dane rynkowe kryptowalut dla Bitcoina, Ethereum i Solany z publicznego REST API CoinGecko v3 i zapisuje odpowiedź jako sformatowany plik JSON.

Stanowi warstwę kolekcji danych dla projektu akademickiego z przedmiotu Bazy Danych — zebrane rekordy JSON są materiałem źródłowym do projektowania i zasilania schematu relacyjnej bazy danych.

Endpoint API:
https://api.coingecko.com/api/v3/coins/markets

Plik wyjściowy:
coingecko_response.json — tworzony / nadpisywany w bieżącym katalogu

💡
Typowe użycie: python main.py
Skrypt jest przeznaczony do uruchamiania bezpośrednio z CLI. Może być też importowany i wywoływany jako from main import main; main().

Funkcje

def main() → None public
Pobiera dane rynkowe kryptowalut z CoinGecko i zapisuje je lokalnie.

Wysyła pojedyncze żądanie GET do endpointu /coins/markets publicznego API CoinGecko v3, żądając statystyk rynkowych wyrażonych w USD dla Bitcoina, Ethereum i Solany. Surowa odpowiedź JSON jest formatowana (indent=4) i zapisywana do pliku coingecko_response.json w bieżącym katalogu roboczym. Nazwy pól pierwszego rekordu są wypisywane na stdout w celach inspekcyjnych.
⚙

Parametry zapytania HTTP

Poniższe parametry są przekazywane jako query string do endpointu GET /api/v3/coins/markets:

Parametr Wartość Opis
vs_currency "usd" Wszystkie wartości monetarne wyrażone w dolarach amerykańskich
ids "bitcoin,ethereum,solana" Identyfikatory monet do pobrania (CoinGecko IDs)
order "market_cap_desc" Sortowanie wyników malejąco po kapitalizacji rynkowej
per_page 10 Maksymalna liczba rekordów na stronę paginacji
page 1 Numer strony (pobieramy zawsze pierwszą)
sparkline "false" Pomija dane sparkline — 7-dniowy mini-wykres cen
price_change_percentage "24h,7d" Dodaje pola ze zmianami procentowymi za 24h i 7 dni
⚡

Efekty uboczne

🌐 Wykonuje jedno wychodzące żądanie HTTP GET z timeoutem 10 sekund do api.coingecko.com
💾 Tworzy lub nadpisuje plik coingecko_response.json w bieżącym katalogu roboczym (sformatowany JSON, indent=4, ensure_ascii=False)
🖨 Wypisuje na stdout: kod statusu HTTP, ścieżkę pliku wyjściowego, listę nazw pól JSON z pierwszego rekordu
⚠️
Uwaga: każde kolejne wywołanie nadpisuje coingecko_response.json. Dane historyczne nie są zachowywane — w planowanej wersji każdy snapshot będzie zapisywany do bazy danych z sygnaturą czasową.

Przykład działania

Kod źródłowy

import json
from pathlib import Path
import requests


def main() -> None:
    url = "https://api.coingecko.com/api/v3/coins/markets"

    params = {
        "vs_currency": "usd",
        "ids": "bitcoin,ethereum,solana",
        "order": "market_cap_desc",
        "per_page": 10,
        "page": 1,
        "sparkline": "false",
        "price_change_percentage": "24h,7d",
    }

    response = requests.get(url, params=params, timeout=10)
    print("Status code:", response.status_code)
    response.raise_for_status()

    data = response.json()

    output_path = Path("coingecko_response.json")
    output_path.write_text(
        json.dumps(data, indent=4, ensure_ascii=False),
        encoding="utf-8",
    )

    print(f"Zapisano dane do pliku: {output_path}")

    if data:
        print("\nDostępne pola dla pierwszego rekordu:")
        for key in data[0].keys():
            print("-", key)


if __name__ == "__main__":
    main()
