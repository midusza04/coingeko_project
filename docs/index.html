app.py — główne funkcje
app.py — baza danych + ETL + UI główny moduł
create_database() Tworzy 3 tabele, indeksy i wstawia listę 5 monet. Wywoływana przy starcie. database
fetch_market_chart() GET /coins/{id}/market_chart — pobiera 365-dniową historię. api
store_market_chart() INSERT OR REPLACE do market_snapshots — idempotentny zapis historyczny. etl
fetch_markets_current() GET /coins/markets — bieżący snapshot 5 monet. api
store_current() INSERT do market_current — append-only, z collected_at UTC. etl
load_snapshots() SELECT + JOIN → pandas DataFrame. Cache 60s (@st.cache_data). read
page_*() × 6 Overview · Data Collection · Time Series · Quantitative · Dashboard · Correlation ui
Schemat bazy danych (crypto_market.db)
cryptocurrencies id TEXT PK · symbol · name (5 wierszy)
market_snapshots record_id PK · crypto_id FK · snapshot_date · price_usd · market_cap · total_volume
  UNIQUE(crypto_id, snapshot_date) · INDEX idx_snapshots_date (~1 826 wierszy)
market_current record_id PK · crypto_id FK · collected_at · price · mcap · high_24h · low_24h · changes · rank · supply · ath
  INDEX idx_current_collected_at (append-only)
